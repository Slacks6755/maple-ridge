# Maple Ridge Construction — mapleridgedev.com

Marketing/lead-gen site for Maple Ridge Construction & Development (owner: Zack Newman), a licensed general contractor in Etowah, TN serving McMinn, Bradley, Polk & Monroe Counties. The site's job is to turn visitors into leads: call, text, email, or the contact form.

## Stack & commands

- Astro 5, fully static output. Tailwind CSS 3 (`@astrojs/tailwind`, `applyBaseStyles: false` — base styles live in `src/styles/globals.css`). Icons via `astro-icon` + Lucide (`lucide:*`). Images via Astro assets + sharp.
- `npm run dev` / `npm run build` (→ `dist/`) / `npm run preview`
- `npm run convert-photos` — converts/resizes new project photos (see `scripts/convert-photos.mjs`)
- Deploy: push to `main` → Netlify builds and publishes `dist/`. Config in `netlify.toml`.

## URL rules (important — this broke Search Console once)

**No internal URL ends with a trailing slash** (the root `/` is the only exception). Pages build as flat files (`build.format: 'file'` → `about.html`), which Netlify serves at `/about` and 301s `/about/` down to. Any slash URL in links, canonicals, or the sitemap registers as a redirect in Google. This matches the original Next.js site's URL form, so old indexed URLs and backlinks resolve without redirects.

- `trailingSlash: 'never'` + `build: { format: 'file' }` are set in `astro.config.mjs`; dev 404s slash URLs on purpose.
- Canonical + og:url are computed in `BaseLayout.astro` from the pathname — never pass a canonical manually.
- Sitemap is generated by `src/pages/sitemap.xml.ts` (plain `<loc>` entries only). Keep it in sync with the page list.
- `netlify.toml` handles: non-www → www, old `/service-areas/:area/:service` combo pages → city page, old `/services` hub → home.

## Structure

- `src/pages/` — the full page set: home, about, contact, projects (single-page portfolio), 3 service pages (`services/*.astro`, static files, not a dynamic route), service-areas index + `[slug]` city pages, 404, `sitemap.xml.ts`.
- `src/data/` — single source of truth; no hardcoded contact info or service lists in pages:
  - `contact.ts` — phone/email/address/hours/owner. Use `contact.phoneHref` for `tel:`, `` `sms:${contact.phoneRaw}` `` for texts.
  - `services.ts` — 3 services (New Construction, Remodeling, Land Development) with slugs + hrefs.
  - `service-areas.ts` — 5 city pages (Etowah, Athens, Cleveland, Madisonville, Sweetwater) with local content + FAQs. Polk County is served but has no city page. Adding an area here auto-generates its page, nav links, and sitemap entry.
  - `projects.ts` — portfolio metadata; photos live in `src/assets/projects/<slug>/` (optionally `before/`/`after/` subdirs), matched by `src/lib/projects.ts` glob. Some category/location fields are guesses pending Zack's confirmation.
- `src/layouts/BaseLayout.astro` — head/SEO/GA + Header, Footer, FloatingContact, LocalBusinessSchema on every page.
- `src/components/FloatingContact.astro` — mobile-only floating Call/Text pills, bottom-right. On desktop the header's green top strip carries call/text instead (`hidden lg:block`); don't show both on the same breakpoint.

## SEO / analytics

- JSON-LD: `LocalBusinessSchema.astro` sitewide; FAQPage schema inline on service + service-area pages.
- Google Analytics 4 (`G-NEE25MJKMQ`) in BaseLayout. Contact CTAs carry `data-track="..."` attributes; a delegated listener in BaseLayout fires GA events — add `data-track` to any new call/text/email/form CTA.
- `public/robots.txt` and `public/llms.txt` are static — update llms.txt when pages change.

## Contact form

Netlify Forms (`data-netlify`, form name `contact`, honeypot `bot-field`). Submissions land in the Netlify dashboard; the email notification to zack@mapleridgedev.com must be configured in the dashboard — code alone does not deliver leads.

## Client preferences (from Zack)

- No years-of-experience claims or unverifiable promises anywhere on the site.
- Tone: honest, direct, rural Tennessee. Not corporate, not AI-sounding — Zack attributes past Google indexing trouble partly to AI-sounding filler pages.
- Portfolio stays one page with before/after galleries — no per-project pages.
- No PWA, no blog. Fewer, better pages beat more pages.
